# Шаманка — Telegram-бот раскладов Таро

## Что это
Telegram-бот: пользователь пишет вопрос → бот сам определяет тип запроса →
выбирает подходящий расклад → случайно тянет карты → присылает картинки карт
и текстовую трактовку от нейросети.

## Стек
- Python 3.12, aiogram 3 (FSM для онбординга)
- PostgreSQL (профили, история раскладов, счётчик бесплатных)
- LLM через OpenRouter (модель Gemini) — слой изолирован в отдельном модуле
- Docker + docker-compose
- MVP на long polling (не webhook)


## Ключевые правила проекта

### Классификация — ВСЕГДА однозначная
Один запрос → ровно один расклад. Механизм: правила в classification.yaml
проверяются по приоритету сверху вниз, первое совпадение побеждает.
Приоритеты: 1) формат вопроса (да/нет, выбор А/Б) → 2) имя+чувство →
3) узкие темы → 4) общие сферы → 5) универсальный fallback.
Никогда не задавать пользователю уточняющих вопросов о выборе расклада.

### Позиции раскладов — ФИКСИРОВАННЫЕ
Позиции берутся ТОЛЬКО из spreads.yaml. Нейросеть их НЕ придумывает —
это гарантирует узнаваемость фирменных раскладов ("Вокзал для двоих"
всегда одинаковый). Нейросеть наполняет позиции смыслом выпавших карт,
но не меняет сами позиции.

### Тяга карт
- Случайный выбор нужного числа карт из 78 без повторов в одном раскладе
- Использовать модуль secrets (криптостойкий), не random
- Ориентация карты (прямая/перевёрнутая) — 50/50 случайно
- Результат тяги фиксируется в БД ДО генерации текста

### LLM изолирован
Весь код обращения к модели — только в src/llm.py. Две функции:
classify(question) → spread_id, interpret(context) → текст трактовки.
Ключ из .env (OPENROUTER_API_KEY). Промпты тестируются на прод-модели (Gemini),
не на Claude — текст на разных моделях звучит по-разному.

### Персона и тон (см. spreads_source.md ч.5, эталон — prompts/interpret.md)
- Голос — **Шаманка**: древняя мудрая ведунья из вселенной Дюны, сидит у костра.
  Женский, тёплый, но с достоинством; поэтично-метафоричный (стихии, луна, нити,
  дым, дороги), густой эзотерический вайб. На «ты», к женщине 24–45.
- **Ответ дроблёный:** 2–4 коротких фрагмента (одна мысль = одно сообщение),
  разделитель `---`. reading.py режет ответ по нему и шлёт несколькими сообщениями.
  (Прежнее требование «одно сообщение» этим заменено.)
- В раскладах НЕ здороваться штампом — начинать с образа. Онбординг: короткая
  легенда от лица Шаманки. Плашки 18+ НЕТ. Дисклеймеров (мед./фин./юр.) НЕТ.
- Эмодзи только служебные и скупо (🌙 ✨ 🔥 🌊 🌿, круги/стрелки/галочки); смайлов
  с лицами НЕТ.
- Сохранённая безопасность: не предрекать буквально смерть, тяжёлую болезнь,
  катастрофу. Башня/Смерть/Дьявол = порог, слом старого и рождение нового.

## Монетизация (заложить в схему, включить позже)
Счётчик free_readings в таблице users. Пейволл = проверка перед тягой карт.
Оплата ТОЛЬКО через Telegram Stars (правила платформы для цифровых услуг).
На MVP не активна.
