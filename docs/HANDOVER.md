# Передача проекта «Прорицательница» (Telegram-бот раскладов Таро)

_Обновлено: 13 июля 2026. Документ для передачи проекта: как всё устроено,
где что лежит, как деплоить и что держать в голове. Секретов здесь нет —
только указания, где они живут._

## 1. Что это

Telegram-бот **@dune_tarrot_bot**: пользователь задаёт вопрос → бот сам выбирает
подходящий расклад (LLM-классификатор по правилам из YAML) → тянет карты
(криптостойко, без повторов) → присылает альбом картинок карт + образную
трактовку от нейросети в голосе «Прорицательницы» (ведунья у костра из вселенной Дюны).

- Репозиторий: `andrewkrrupnov-cpu/shamanka` (GitHub, private)
- Прод: VPS reg.ru (рег.облако) `194.67.116.78`, Ubuntu 22.04, Docker Compose
- Код на проде: `/opt/shamanka`
- Локальная рабочая копия: `~/Downloads/shamanka`

## 2. Стек

- Python 3.12, aiogram 3 (long polling, FSM)
- PostgreSQL 16 (профили, история раскладов, промокоды)
- LLM через OpenRouter: `google/gemini-2.5-flash` (трактовки),
  `google/gemini-2.5-flash-lite` (классификация и валидация вопроса)
- Docker + docker-compose (сервисы `bot` + `db`)
- Оплата: Telegram Stars (XTR); авто-переключение на ЮKassa при появлении токена

## 4. Ключевые правила (НЕ нарушать)

1. **Позиции раскладов фиксированы** — только из `spreads.yaml`. Нейросеть
   наполняет их смыслом, но не меняет.
2. **Классификация однозначна** — правила из `classification.yaml` по приоритету,
   первое совпадение побеждает. Никаких уточняющих вопросов пользователю.
3. **Все обращения к модели — только через `src/llm.py`** (classify живёт в
   classifier.py, но транспорт — `openrouter_chat` из llm.py).
4. **Тяга карт** — модуль `secrets`, без повторов, ориентация 50/50; результат
   пишется в БД **до** генерации текста.
5. **Безопасность контента**: не предрекать буквально смерть/болезнь/катастрофу.
   Башня/Смерть/Дьявол = трансформация. (Дисклеймеров и 18+ нет — решение владельца.)

## 5. Персона и формат ответа

- Голос — **Прорицательница**: ведунья у костра, образно, но КОНКРЕТНО (образ всегда
  расшифровывается в практический смысл). На «ты», род по полу пользователя
  (муж./жен., из онбординга). Не здоровается. Смайлов-лиц нет.
- Формат расклада: альбом карт (перевёрнутые повёрнуты на 180°) со списком карт
  в подписи → по блоку на каждую карту с жирным заголовком
  `<эмодзи> Позиция – Карта (перевёрнутая)` → финал `🔮 Итог` (связывает все
  карты + конкретный совет). Блоки разделены `---` → отдельные сообщения.
- Эталон — `prompts/interpret.md`. Менять тон = менять этот файл.

## 6. Функциональность

| Фича | Как работает |
|---|---|
| Онбординг | /start → легенда → имя → пол (кнопки). «🔄 Обновить имя» в профиле — повторный проход |
| Расклад | ТОЛЬКО по кнопке «🔮 Сделать расклад» → FSM ждёт вопрос → валидация (билиберда отклоняется) → classify → draw → БД → interpret → альбом + блоки. 1-й бесплатно |
| Уточнения | После трактовки — 2 контекстных вопроса инлайн-кнопками (`llm.suggest_clarifications`, дешёвая модель). Тап (`callback clr:{id}:{idx}`) → ответ по ТЕМ ЖЕ картам (`interpret_clarification`), без новой тяги. Бесплатно, макс. 2 на расклад; хранится в `readings.clarifications`. Обработчик — `reading.on_clarify` |
| Баланс | `users.free_readings`, списание после успешной генерации; при 0 — витрина покупки |
| Карта дня | Кнопка — бесплатно; карта детерминирована на (юзер, дата). Подписка 🔔/🔕 + рассылка 06:00 UTC (`daily_loop`) |
| Покупка | Пакеты: 10 раскладов и 100 раскладов. Stars: 50⭐/150⭐; рубли (при боевом `YOOKASSA_PROVIDER_TOKEN`): 99 ₽/299 ₽. ⚠️ Мелкий пакет «3 расклада» убран: минимум рублёвого платежа у Telegram — 87,73 ₽ (`min_amount=8773`), иначе `CURRENCY_TOTAL_AMOUNT_INVALID`. Чеки 54-ФЗ: `YOOKASSA_SEND_RECEIPT=1` → инвойс запрашивает email и шлёт состав чека в `provider_data` (обязательно при автоотправке чеков в ЛК ЮKassa) |
| Подарок | 🎁 150⭐/299₽ → одноразовый промокод на 100 раскладов + пересылаемое сообщение с кнопкой-диплинком `t.me/bot?start=promo_CODE` |
| Промокоды | 🎟 ввод кода или диплинк; `redeem_promo` атомарный (FOR UPDATE). Флаг `reusable`: платные — одноразовые, владельческие — без лимита |
| Профиль | 👤 имя, остаток раскладов, статус карты дня |
| Меню | Постоянная reply-клавиатура `is_persistent=True` |

**Активные владельческие промокоды:** `Мира` (+100, многоразовый), `KR` (+10,
многоразовый). Управление — вручную через python в контейнере (см. §9).

## 7. Прод и сеть — САМОЕ ВАЖНОЕ

VPS в РФ, поэтому два обходных решения. Оба задокументированы в коде.

1. **Telegram**: провайдер фильтрует часть IP Telegram, DNS отдаёт битый адрес.
   Пин в `docker-compose.yml`: `extra_hosts: api.telegram.org:149.154.167.220`.
   Если бот молчит — первым делом проверить, жив ли этот IP (порт 443).
   Плюс в `bot.py` `delete_webhook` обёрнут в таймаут (первый запрос изредка
   залипает).
2. **OpenRouter**: Cloudflare блокирует РФ-IP (403 «Access denied by security
   policy»). LLM-трафик идёт через покупной SOCKS5-прокси (proxy6.net, Рига):
   env `OPENROUTER_PROXY=socks5h://логин:пароль@IP:порт` в `.env`.
   ⚠️ Прокси платный по сроку — при истечении LLM отвалится с теми же
   симптомами («Дым сегодня густой…» на любой вопрос): продлить или заменить
   строку в .env и пересоздать контейнер.
   WireGuard-вариант НЕ работает: дата-центр режет протокол по DPI.

**Долгосрочная рекомендация:** переезд на европейский VPS (~300–400 ₽/мес)
убирает оба костыля и прокси-платёж.

## 9. Операции

**Деплой** (из локальной копии):
```bash
cd ~/Downloads/shamanka
git push origin main   # код
rsync -az --delete --exclude '.venv' --exclude '.git' --exclude '__pycache__' \
  --exclude '.env.example' --exclude '.DS_Store' \
  ./ root@194.67.116.78:/opt/shamanka/
ssh root@194.67.116.78 'cd /opt/shamanka && docker compose up -d --build'
```
⚠️ После изменения `.env` нужен `docker compose up -d --force-recreate bot` —
простой restart не перечитывает env_file.

**Логи:** `docker compose logs bot --tail 50` (в `/opt/shamanka`).
**БД:** `docker compose exec db psql -U shamanka -d shamanka` (`\d users` и т.п.).
**Промокод завести:** python-снипет через `docker compose exec -T bot python -`
с `db.Promo(code=..., readings=..., reusable=...)`.

**Тесты** (нужен OPENROUTER_API_KEY в окружении, из корня репо):
```bash
python scripts/test_classify.py            # 30 вопросов, ожидание 30/30
python scripts/test_prompts.py --seed 42   # тон/формат трактовок, exit 0
```

## 10. Схема БД (создаётся автоматически на старте)

- `users`: id (tg id), name, gender(male/female/other), free_readings (default 1),
  onboarded, daily_card, created_at. Новые колонки доводятся ALTER'ами в
  `db.create_tables()` (create_all не меняет существующие таблицы).
- `readings`: id, user_id, question, spread_id, spread_title, cards (JSON,
  [["Карта","прямая"],…]), interpretation, clarifications (JSON,
  {"questions":[…],"used":[индексы]}), created_at. Тяга пишется ДО текста.
- `promocodes`: code (PK), readings, reusable, created_by, used_by, used_at.

## 11. Экономика

Себестоимость одного расклада (замерено): **0.19–0.39 ¢** в зависимости от числа
карт (3→12), из них ~90% — трактовка на gemini-2.5-flash. 1000 раскладов ≈ $2.5–4.
Плюс прокси (~150 ₽/мес) и VPS.

## 12. Известные хвосты / бэклог

- FSM-хранилище — MemoryStorage: при рестарте бота теряется состояние «жду
  вопрос» (пользователь просто жмёт кнопку заново). Для надёжности — Redis.
- Утренняя рассылка — цикл в процессе бота: если рестарт случится позже 06:00
  UTC, рассылка этого дня пропускается. Для надёжности — вынести в cron.
- CLAUDE.md всё ещё говорит «22 расклада», фактически 27 (решить: править
  текст или состав).
- Классификатор: «стоит ли + бизнес»-кейсы держатся на смысле, а не строгом
  приоритете — на более слабой модели могут уплыть (усилить note в
  classification.yaml при смене модели).
- `interpret()` синхронный (requests) — в хендлерах обёрнут в asyncio.to_thread;
  при росте нагрузки перейти на aiohttp.
- Возвраты Stars (refundStarPayment) не реализованы.
- Картинка экрана «до Старт» ставится только вручную через @BotFather
  (Edit Description Picture) — Bot API этого не умеет.

## 13. История ключевых решений

- Голос переделывался трижды: мягкая «Шаманка» → властная «Саира» → финальная
  «Шаманка у костра» (образно + конкретно, дроблёные сообщения). Позже персона
  переименована в «Прорицательница» (голос и подача те же).
- «Одно сообщение на трактовку» заменено на «блок на карту + Итог» (по референсу
  владельца), с обязательным разбором каждой карты (тест это проверяет).
- Дисклеймеры (мед., 18+) убраны сознательно; сохранена только внутренняя
  безопасность «не предрекать буквальную смерть/болезнь».
- Оплата: Stars как мост до подключения ЮKassa (скриншоты цен для модерации
  ЮKassa можно делать с текущей витрины).
